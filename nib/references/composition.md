# Composition — the two registers

Every Nib image is made in one of two registers. The methodology is identical
in both (one idea, the avatar performs it, white ground, sparse labels, the
erase test) — the register only sets which **image grammar** is allowed.

- **Editorial** (the default) — one caught scene: the avatar mid-action on one
  or two low-tech built objects, the meaning implied. No diagram machinery.
- **Explainer** — a hand-built sketch-diagram: a few stations, one clear flow
  direction, a couple of callouts — for when the reader must *trace* the
  structure, not just feel it. Still hand-drawn, still on white, still starring
  the avatar; it just draws the structure instead of staging a single move.

**Editorial wins every tie.** Route an image to the explainer register only when:

1. **the user asks for it** — "show the flow", "diagram the pipeline", "map the
   steps", "make it traceable", "as an explainer"; or
2. **the idea itself IS a traceable structure** — its point lives in the
   stations and their connections (a pipeline with named stages, a fan-out, a
   timeline, a loop, a layered stack), and a single caught moment would force
   the reader to take the structure on faith.

A process that is merely *evidence* for a judgment stays editorial — the idea
is a claim, not a structure. Opinions, quotes, launches, and anecdotes stay
editorial. A set may mix registers (an editorial hero over a couple of
explainer anchors is a natural article shape).

## Editorial — turn the idea into a move

Start from the one sentence the picture must land, then find the **physical
move** that embodies it — something the avatar can be mid-action on. Push the
abstract into the concrete: "we ship too slowly" → the avatar cranking a press
that drips a single parcel; "we're buried in inputs" → the avatar bailing a
bucket that keeps overflowing. The move *is* the picture; until the move has a
name, there is no image yet. Give it one or two built things to happen on,
then wedge the avatar into the move — never posed politely beside it.

**Stagings that tend to land** (starting angles, not labels to write on the image):

- **Contraption** — one absurd machine that performs the idea: small input, one output.
- **Change** — the same scene in two states (jumbled → settled, by-hand → automatic).
- **Throughput** — something travels left-to-right and is transformed on the way.
- **Snag** — the whole thing jams at one point, and the avatar is usually the jam.
- **Build-up / drain** — it stacks, fills, leaks, or empties over time.
- **Crossing** — a gap, gate, ramp, or threshold the avatar moves something over.
- **Mini-comic** — 2–4 small panels inside ONE image, read left to right, one
  action per panel; the avatar and the key object carry through every panel so
  it reads as the same moment advancing (stuck → small slice → shipped).

## Explainer — the diagram register

When the structure is the point, draw it as a **hand-built sketch-diagram**, not
a corporate flowchart:

- **3–6 stations only.** Boxes, drums, trays, bins — drawn in the same hand line
  as everything else, never crisp UI rectangles. Name each in 1–3 words.
- **One flow direction.** Left→right or top→down. Use the accent that carries
  *flow* (orange) for the main path's arrows; keep it a single spine, not a web.
- **The avatar is in one key station, doing the load-bearing action** — routing,
  filtering, jamming, switching. The erase test still applies: if the avatar is
  just standing next to the diagram, the structure is decoration of itself —
  put it to work at the station that matters most.
- **A callout or two**, hand-lettered, for the one or two things the reader must
  notice (a bottleneck in red, a feedback edge in blue).
- Still on white, still ≥35% quiet space, still no title block, legend, grid, or
  corner heading — the things that make it read as a slide.

Pick the structure type from the idea: **workflow** (in → process → out),
**fan-out / fan-in**, **before/after**, **timeline / route**, **loop**, or a
**layered stack**. One type per image; don't mix.

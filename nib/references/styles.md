# Looks

One look per piece. Pass it as `--style <id>`. Most read on a white ground;
chalk and phosphor carry their own dark ground (light line on dark).

| id | label | use it for |
|----|-------|------------|
| `marker` | Marker | the default — warm, friendly, hand-drawn explainer feel |
| `riso` | Riso | zine / indie editorial; limited spot colors + grain |
| `blueprint` | Blueprint | systems, architecture, "how it's built" pieces |
| `woodcut` | Woodcut | bold, punchy, high-contrast; strong single statements |
| `pixel` | Pixel | dev / retro / game-adjacent content |
| `clay` | Clay | soft, approachable, product-y warmth |
| `gouache` | Gouache | painterly, premium editorial |
| `chalk` | Chalk | teaching / explainers / walkthroughs (slate board, light line) |
| `diorama` | Diorama | soft, storybook, friendly product warmth |
| `enamel` | Enamel | bold, collectible, glossy pin look; launches, badges |
| `felt` | Felt | tactile, handmade, cozy; approachable how-tos |
| `manila` | Manila | retro office / paperwork / process; dry, deadpan |
| `phosphor` | Phosphor | terminal / retro-tech / systems (dark screen, glowing line) |

Keep one look consistent across a whole article so the set reads as a series.
The look only changes the rendering; the methodology (one idea, the avatar
performs it, the register's grammar, short labels) never changes. For the
editorial-vs-explainer register, see `composition.md`.
